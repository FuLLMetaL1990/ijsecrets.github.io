Ruby on Rails: From zero to deploy

  ejelome /   October 24, 2019 /   development ▪ ruby ▪ ruby-on-rails ▪ web-application-framework

Summary

This document is a densed summary of the Ruby on Rails Tutorial's chapter 1: From zero to deploy. The goal is to focus solely on the project setup and deployment to Heroku.

This great book, Ruby on Rails Tutorial, was finally updated to the 6th edition to cover Ruby on Rails 6. Unlike before, though, all chapters of the book isn't available online for free as it used to, however, in my opinion, people who learned a somewhat complete sense of Rails will still benefit from its first 3 free chapters; one being the first chapter: From zero to deploy — which this document is about.

Prerequisites

Before proceeding, make sure that the following are installed:

  • RVM
  • Yarn

Note: In this guide, we'll use the name rails-fztd (replace it with whatever name you want).

Setup

  1. Create and move to project directory:
    $ \
mkdir -p ~/Projects/
cd ~/Projects/
  2. Create and move to Rails app:
    $ \
rails new rails-fztd
cd rails-fztd/
  3. Run web server:
    $ rails s
    Then visit http://localhost:3000.

Pre-deployment

Project

  1. Update the Gemfile:
    # File: Gemfile
...
# gem 'sqlite3', '~> 1.4'
...
group :development, :test do
  ...
  gem 'sqlite3'
end

group :production do
  gem 'pg'
end
...
    Notes:
    • Move sqlite3 gem to :development, :test groups
    • Create :production group then add pg gem in it
    • Heroku doesn't support SQLite but enforces PostgreSQL (hence pg)
    • Gems (e.g. sqlite3 and pg) without a version will install the latest version
  2. Install postgresql-libs package:
    Note: This package installs the essential shared libraries for PostgreSQL clients.
    $ sudo pacman -S postgresql-libs
  3. Install :development gems:
    $ bundle install --without production
    Note: The --without production parameter excludes installing gems specified in :production group.

Heroku

  1. Sign up (if no account)
  2. Install Heroku CLI:
    $ yay -S heroku-cli
  3. Log in (interactively):
    $ heroku login --interactive
    Note: The --interactive parameter uses the terminal-based login flow instead of opening a web browser.
  4. Create a new Heroku app:
    $ heroku apps:create rails-fztd
    Note: The apps:create parameter allows us to create a Heroku app with a specific name.

Deployment

  1. Stage and commit (local):
    $ \
git add -A
git commit -m 'Initial commit'
  2. Push to Heroku master (remote):
    $ git push heroku master
    Then visit https://rails-fztd.herokuapp.com.
  3. Check Heroku logs for possible errors:
    $ heroku logs

Congratulations! We've finally learned how to deploy our Rails app on Heroku :)
